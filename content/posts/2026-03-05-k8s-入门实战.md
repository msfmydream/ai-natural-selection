说实话，我搞了三天才把第一个K8s应用跑起来。

之前一直在用Docker Compose，觉得挺香的。直到上周领导说新项目要上K8s，让我先调研一下。我想着这不就是容器编排嘛，应该很快上手。结果... 是我太年轻了。

## 第一天：连环境都装不好

### minikube 安装踩坑

官方文档说Mac用brew装就行：

```bash
brew install minikube
```

装是装上了，启动的时候报错：

```
❌  Exiting due to MK_USAGE: No IP address provided
```

Google了半天，发现是Docker Desktop的问题。重启Docker，再试，又报错：

```
❌  Exiting due to GUEST_PROVISION: Failed to start host
```

最后发现是权限问题，加上sudo就好了（虽然官方不推荐，但我是真的没办法了）：

```bash
sudo minikube start --driver=docker
```

### kubectl 版本不匹配

装完minikube，kubectl也有坑。brew装的kubectl版本太新，跟minikube自带的K8s版本对不上，执行命令老报warning。

解决办法是用minikube自带的kubectl：

```bash
alias kubectl="minikube kubectl --"
```

或者直接用docker exec进minikube的容器里操作，虽然笨了点，但至少不会报错了。

## 第二天：终于跑起来第一个Pod

跟着官方文档写了个Deployment：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
```

apply之后看状态：

```bash
kubectl get pods
```

全是`ImagePullBackOff`。我以为是公司网络问题，换了个镜像，还是不行。

最后发现是`nginx:latest`这个tag在minikube里拉不下来（可能是被墙了，也可能是Docker Hub的rate limit）。换成`nginx:alpine`就好了。

**教训**：不要用latest tag，在K8s里尤其容易出问题。

## 第三天：Service访问不通

Pod跑起来了，但Service怎么都访问不了。

我按文档创建了NodePort类型的Service：

```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: NodePort
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
    nodePort: 30080
```

然后想当然地用`localhost:30080`去访问，结果连不上。

查了半天资料才知道，minikube的NodeIP不是localhost，要这样获取：

```bash
minikube ip
```

然后用那个IP访问才行。或者直接用minikube的service命令：

```bash
minikube service nginx-service
```

会自动打开浏览器访问。

## 现在能跑起来的完整配置

经过三天踩坑，这套配置在我环境里是能跑的：

```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: web-app
  template:
    metadata:
      labels:
        app: web-app
    spec:
      containers:
      - name: nginx
        image: nginx:alpine  # 别用latest
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: web-service
spec:
  type: NodePort
  selector:
    app: web-app
  ports:
  - port: 80
    targetPort: 80
    nodePort: 30080
```

```bash
# 部署
kubectl apply -f deployment.yaml

# 查看状态
kubectl get pods
kubectl get svc

# 访问（Mac/Linux）
curl http://$(minikube ip):30080

# 或者让minikube自动打开
minikube service web-service
```

## 一些个人感悟

1. **本地开发和生产环境差别很大**：minikube只是个玩具，很多生产环境的问题（网络策略、存储、多节点调度）根本模拟不了。

2. **文档坑很多**：官方文档默认你什么环境都是完美的，实际上国内网络、公司内网代理这些坑一个都不会少。

3. **kubectl的语法很反人类**：delete一个资源要`kubectl delete deployment xxx`，为什么不直接`kubectl rm`？
