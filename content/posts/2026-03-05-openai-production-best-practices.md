这份指南提供了一套全面的最佳实践，帮助你从原型过渡到生产环境。无论你是经验丰富的机器学习工程师还是刚刚入门的爱好者，这份指南都能为你提供成功部署应用所需的工具——从保护 API 访问到设计能够处理高流量的稳健架构。

## 组织管理

登录 OpenAI 账户后，你可以在组织设置中找到组织名称和 ID。组织名称是用户界面中显示的组织标签，组织 ID 是可在 API 请求中使用的唯一标识符。

属于多个组织的用户可以通过传递 header 来指定哪个组织用于 API 请求。这些 API 请求的使用量将计入指定组织的配额。如果未提供 header，则将向默认组织计费。你可以在用户设置中更改默认组织。

### 成员权限管理

你可以从团队页面邀请新成员加入组织。成员可以是**读者**或**所有者**：

**读者权限**：
- 可以进行 API 请求
- 可以查看基本的组织信息
- 可以创建、更新和删除组织中的资源（如 Assistants），除非另有说明

**所有者权限**：
- 拥有读者的所有权限
- 可以修改账单信息
- 可以管理组织内的成员

### 账单限额管理

输入账单信息后，OpenAI 会设置每月 100 美元的使用限额。随着你在平台上使用量的增加，从一级使用层迁移到另一级，你的配额限制会自动提高。你可以在账户设置的限额页面查看当前的使用限制。

如果你想在使用量超过特定金额时收到通知，可以通过使用限额页面设置通知阈值。

## 项目分阶段管理

随着规模扩大，你可能需要为 staging 和 production 环境创建单独的项目。你可以在仪表板中创建这些项目，从而隔离开发和测试工作，避免意外干扰线上应用。你还可以限制用户对生产项目的访问，并为每个项目设置自定义的费率和支出限制。

## 扩展架构设计

当设计用于生产的应用程序或服务时，重要的是考虑如何扩展以满足流量需求。无论选择哪家云服务提供商，有几个关键领域需要考虑：

### 水平扩展

你可能需要水平扩展应用程序以容纳来自多个来源的请求。这可能涉及部署额外的服务器或容器来分配负载。如果选择这种类型的扩展，请确保你的架构设计能够处理多个节点，并且有机制在它们之间平衡负载。

### 垂直扩展

另一种选择是垂直扩展应用程序，这意味着你可以增强单个节点的可用资源。这将涉及升级服务器的能力以处理额外的负载。如果选择这种类型的扩展，请确保你的应用程序设计能够利用这些额外资源。

### 缓存策略

通过存储经常访问的数据，你可以改善响应时间，而无需重复调用我们的 API。你的应用程序需要设计为尽可能使用缓存数据，并在添加新信息时使缓存失效。

有几种不同的方式可以实现这一点。例如，你可以将数据存储在数据库、文件系统或内存缓存中，具体取决于对你的应用程序最有意义的方式。

### 负载均衡

考虑使用负载均衡技术来确保请求均匀分布在可用服务器之间。这可能涉及在服务器前使用负载均衡器或使用 DNS 轮询。平衡负载将有助于提高性能并减少瓶颈。

## 延迟优化策略

### 模型选择

我们的 API 提供不同级别的复杂性和通用性的模型。最强大的模型，如 `gpt-5`，可以生成更复杂和多样化的完成，但它们也需要更长的时间来处理你的查询。

像 `gpt-5-nano` 这样的模型可以生成更快、更便宜的响应，但它们可能为你的查询生成准确性或相关性较低的结果。你可以选择最适合你用例的模型，并在速度、成本和质量之间进行权衡。

### Token 数量控制

请求大量生成的 token 可能导致延迟增加：

- **降低 max tokens**：对于生成 token 数量相似的请求，具有较低 `max_tokens` 参数的请求延迟更低
- **包含停止序列**：为了防止生成不需要的 token，添加停止序列。例如，你可以使用停止序列生成具有特定数量项目的列表
- **减少生成数量**：尽可能降低 `n` 和 `best_of` 的值

### 流式传输

在请求中设置 `stream: true` 使模型在 token 可用时立即开始返回，而不是等待完整的 token 序列生成完毕。这不会改变获取所有 token 的时间，但对于我们希望显示部分进度或准备停止生成的应用程序，它减少了首个 token 的时间。

### 批处理

根据你的用例，批处理可能有帮助。如果你向同一端点发送多个请求，可以将提示批量发送到同一请求中。这将减少你需要发出的请求数量。`prompt` 参数最多可容纳 20 个唯一提示。我们建议测试此方法，看看它是否有帮助。

## 安全与合规

将原型迁移到生产环境时，你需要评估和解决可能适用于应用程序的任何安全和合规要求。这将涉及检查你处理的数据、了解我们的 API 如何处理数据，以及确定你必须遵守的法规。

一些常见领域包括数据存储、数据传输和数据保留。你可能还需要实施数据隐私保护措施，例如尽可能进行加密或匿名化。此外，你应该遵循安全编码的最佳实践，如输入清理和适当的错误处理。
