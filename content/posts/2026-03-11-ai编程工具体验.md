用 Copilot 和 Cursor 写代码有一个多月了，分享一下真实感受，不吹不黑。

## 我的使用场景

- **日常开发**：Java 后端，Spring Boot 那一套
- **脚本工具**：Python 写数据处理脚本
- **学习新技术**：看源码、写 demo

## GitHub Copilot 体验

### 好用的场景

**1. 写样板代码**

比如 Getter/Setter、构造函数、日志打印这些，Copilot 基本能猜出来我要写什么，按 Tab 就行。

```java
// 我写：
private static final Logger logger = 

// Copilot 补全：
private static final Logger logger = LoggerFactory.getLogger(MyClass.class);
```

**2. 写单元测试**

选中一个方法，Copilot 能生成对应的测试用例，虽然不完全对，但改改就能用，省了不少时间。

**3. 写注释文档**

写函数的时候，先写注释描述功能，Copilot 能根据注释生成代码，而且通常还挺准。

```java
/**
 * 根据用户ID查询订单列表，按创建时间倒序
 */
// Copilot 自动补全方法体
```

### 不好用的场景

**1. 复杂业务逻辑**

涉及到多个系统交互、复杂的业务规则，Copilot 生成的代码通常是错的，不如自己写。

**2. 新项目初期**

代码量少，上下文不足，Copilot 补全的准确率很低。等代码量上来了，它才能学会项目的风格。

**3. 不常见的语言/框架**

比如公司内部的私有框架，Copilot 完全不会，只能当普通 IDE 用。

### 使用技巧

1. **函数名要起好**：Copilot 很大程度上是根据函数名和上下文猜意图
2. **多给示例**：写几个例子，Copilot 就能学会模式
3. **别盲目接受**：生成的代码一定要看一遍，我见过它生成过有 bug 的代码

## Cursor 体验
n
Cursor 是最近才开始用的，用了两周。

### 喜欢的功能

**1. Ctrl+K 生成代码**

用自然语言描述需求，Cursor 直接生成代码。比如：

```
写一个 FastAPI 接口，接收用户ID，返回用户信息，
用 SQLAlchemy 查 PostgreSQL
```

它能生成完整的代码，包括模型定义、接口、错误处理，基本能跑。

**2. Ctrl+L 问问题**

看开源项目源码的时候特别有用。直接问：

```
这个函数是做什么的？
这个项目的架构是怎样的？
```

Cursor 会基于整个代码库回答，比自己翻文档快多了。

**3. 代码重构**

选中一段烂代码，让它重构，通常能给出一个更清晰的版本。

```
重构这段代码，用策略模式替代 if-else
```

### 不太行的功能

**1. 大文件处理**

文件超过几千行，Cursor 就会变慢，甚至卡死。

**2. 复杂项目理解**

虽然号称能理解整个代码库，但对于那种几十万行、模块众多的项目，它也会懵。

**3. 中文支持**

有时候用中文提问，回答的质量不如英文。

## 效率提升

| 任务 | 以前 | 现在 | 提升 |
|------|------|------|------|
| 写 CRUD 接口 | 2小时 | 30分钟 | 4x |
| 写单元测试 | 1小时 | 15分钟 | 4x |
| 重构代码 | 2小时 | 1小时 | 2x |
| 理解开源项目 | 1天 | 半天 | 2x |

整体感觉写代码速度快了 2-3 倍，但前提是：

1. 任务不能太复杂
2. 要能看懂 AI 生成的代码
3. 会写 Prompt

## 一些坑

**1. 过度依赖**

刚开始用的时候，我写代码的本能都在退化——脑子里想的不是怎么实现，而是怎么描述给 AI。

后来发现这样不行，AI 不能替代思考，它只能加速实现。

**2. 生成的代码有 bug**

特别是边界情况，AI 考虑不周。有一次它生成的代码没处理空指针，上线后报错了。

**3. 代码风格不一致**

AI 生成的代码风格跟项目不一致，需要手动调整，或者用工具格式化。

## 我的使用 workflow

现在的工作流大概是：

```
1. 需求分析（自己思考）
2. 技术方案（自己设计 + 用 Cursor 验证）
3. 写代码框架（Cursor Ctrl+K）
4. 填充细节（Copilot 补全）
5. 代码审查（自己看一遍 + Cursor 辅助）
6. 写测试（Copilot 生成 + 自己改）
```

AI 负责 60% 的编码工作，我负责 40% 的思考+审查。

## 值不值得买

- **Copilot**：$10/月，值得。写代码确实快了不少，特别是样板代码。
- **Cursor**：免费版够用，Pro 版 $20/月，看需求。如果经常需要理解大型代码库，可以买。

## 总结

AI 编程工具确实能提高效率，但不是银弹。

- 适合：样板代码、单元测试、学习新技术、代码重构
- 不适合：复杂业务逻辑、架构设计、创造性工作

最重要的是，不要放弃思考。AI 是工具，不是替代品。
